MCP Dynamic OAuth Prompt Template

Template master-prompt siap pakai untuk mengintegrasikan server MCP dengan support Dynamic Client Registration (DCR) di proyek baru.

Cara Menggunakan:

  1. Salin prompt di bawah ini dengan menekan tombol "Salin Prompt".
  2. Tempelkan ke dalam chat AI Assistant (Gemini, Claude, Cursor, dll.) di proyek baru Anda.
  3. Lakukan kustomisasi pada bagian akhir prompt (sesuaikan framework, basis data, dan list tools yang Anda inginkan).
mcp-oauth-dcr-prompt-template.txt 
Saya ingin membuat/mengintegrasikan sebuah server Model Context Protocol (MCP) berbasis HTTP (SSE/Streamable HTTP) ke dalam proyek ini. 

Agar integrasi dengan host AI client (seperti ChatGPT) berjalan "seamless" (One-Link Solution) tanpa mewajibkan pengguna menginput Client ID & Client Secret secara manual pada form konfigurasi awal (Host akan mendaftarkan dirinya secara otomatis), kita harus mengimplementasikan standar **Dynamic Client Registration (DCR / RFC 7591)** dan **OAuth Discovery Metadata**.

Tolong buatkan kode dan modul backend yang rapi dengan spesifikasi berikut:

### 1. Endpoint Server MCP & Lazy Auth
- Buat endpoint penanganan SSE (Server-Sent Events) di path `/api/mcp` untuk menerima command JSON-RPC dari client MCP.
- Terapkan mekanisme **Lazy Auth**: client MCP dapat terhubung ke tool publik tanpa autentikasi di awal.
- Bila client memanggil tool yang bersifat proteksi/pribadi (protected tools) tanpa menyertakan token Bearer valid di header `Authorization`, sistem harus menolak dan mengembalikan:
  * Status HTTP: **401 Unauthorized**
  * Header HTTP `WWW-Authenticate`: `Bearer realm="MCP Services", resource_metadata="[BASE_URL]/.well-known/oauth-protected-resource"`
  * JSON body error RPC yang berisi metadata `login_url` menuju halaman login consent.
- Saat client melakukan koneksi pertama kali (`GET` handshake), sertakan header HTTP:
  `Link: <[BASE_URL]/.well-known/oauth-protected-resource>; rel="configuration"`

### 2. File Metadata Discovery (.well-known)
Sistem harus menyajikan dua endpoint publik di path standard berikut (sesuaikan mapping routing dengan framework Anda):
- **`GET /.well-known/oauth-protected-resource`**
  Mengembalikan JSON metadata resource:
  ```json
  {
    "resource": "[BASE_URL]",
    "authorization_servers": ["[BASE_URL]"],
    "resource_name": "My MCP App"
  }
  ```
- **`GET /.well-known/oauth-authorization-server`** (atau `/.well-known/openid-configuration`)
  Mengembalikan metadata server otorisasi:
  ```json
  {
    "issuer": "[BASE_URL]/",
    "authorization_endpoint": "[BASE_URL]/oauth/authorize",
    "token_endpoint": "[BASE_URL]/api/oauth/token",
    "registration_endpoint": "[BASE_URL]/api/oauth/register",
    "response_types_supported": ["code"],
    "code_challenge_methods_supported": ["S256"],
    "grant_types_supported": ["authorization_code", "refresh_token"],
    "token_endpoint_auth_methods_supported": ["client_secret_post", "none"]
  }
  ```

### 3. Dynamic Client Registration (DCR) Endpoint
- **`POST /api/oauth/register`** (atau path registrasi lainnya sesuai rancangan)
  Keleluasaan bagi AI client untuk mendaftarkan dirinya. Endpoint ini mendeteksi payload registrasi dari AI Client (seperti nama client dan redirect URIs) lalu mengembalikan respon JSON berisi Client ID yang valid:
  ```json
  {
    "client_id": "mcp-default-client",
    "client_name": "[NAMA_CLIENT_DARI_REQUEST]",
    "redirect_uris": ["[REDIRECT_URIS_DARI_REQUEST]"],
    "grant_types": ["authorization_code", "refresh_token"],
    "response_types": ["code"],
    "token_endpoint_auth_method": "none"
  }
  ```

### 4. Alur Otorisasi & Pertukaran Token (OAuth Flow)
- Buat halaman/route **Consent / Authorization Endpoint** (`/oauth/authorize`):
  * Menerima parameter `client_id`, `redirect_uri`, dan `code_challenge`.
  * Meminta pengguna login/konfirmasi akses. Keberhasilan login menghasilkan token otorisasi (`code`) berumur pendek.
- Buat **Token Endpoint** (`/api/oauth/token`):
  * Menerima penukaran token `code` menjadi Bearer access token dan refresh token.
